# 技术选型

> 基于需求分析和系统设计讨论后确定的最终技术栈。

---

## 选型总览

| 决策项 | 选择 | 说明 |
|--------|------|------|
| 后端语言/框架 | **Python + FastAPI** | 开发效率高，异步支持好，Pydantic 模型校验与 API 文档自动生成契合项目需求 |
| 数据库 | **SQLite**（开发阶段） | 零配置、文件级，适合开发/演示。后续生产可迁移至 PostgreSQL |
| ORM | **SQLModel** | FastAPI 作者作品，DB 模型与 API Schema 合一，减少重复代码。底层为 SQLAlchemy，复杂查询可降级使用 |
| 缓存/分布式锁 | **进程内内存**（asyncio.Lock + dict） | 零依赖。单实例够用，后续水平扩展时切 Redis |
| 前端框架 | **Vue 3 + Element Plus** | 国内生态丰富，组件库完善 |
| 部署方式 | **本地双进程**（Uvicorn + Vite Dev Server） | 前后端分离，各自独立启动 |
| 定时任务调度 | **asyncio.create_task 后台协程** | 零依赖，一个 while True 循环实现轮询调度 |
| JWT 鉴权 | **FastAPI OAuth2PasswordBearer + python-jose** | 官方推荐方案，与 Depends 依赖注入天然集成 |
| API 文档 | **FastAPI 内置 Swagger (/docs)** | 自动生成，一期前端靠此测试接口 |
| 密码哈希 | **passlib + bcrypt** | 标准方案 |

## 一期范围

**纯后端 API 开发，前端通过 Swagger (/docs) 测试验证。**

优先完成的模块顺序：
1. 基础设施（数据库、鉴权、配置）
2. 账号模块（注册、登录、用户信息）
3. 充电桩模块（桩管理、状态查询）
4. 充电会话模块（发起、轮询、修改、取消、停止）
5. 账单模块（生成、查询、支付）
6. 调度系统（轮询调度、三区流转）
7. 管理端（配置、报表、队列管理）
